  • Supports adding new tools without modifying agent code

  • Execution planning
  • State management
  • Error handling

Implements ReAct pattern with multi-step reasoning and self-reflection.

  • Thought process
  • Action selection
  • Observation parsing
  • Reflection

Dual-layer memory combining context with persistent knowledge.

  • Long-term storage
  • Semantic search
  • Consolidation

Protocol implementation for discovering and invoking tools.

Complete Data Flow

End-to-end journey of a task through the system

📌 Task Submission → Execution → Completion Flow:

① User submits task via REST API or CLI
② FastAPI validates and routes to Agent Service
③ Agent Core initializes and creates execution plan
④ Reasoning Engine starts ReAct loop
⑤ Agent queries MCP Server for available tools
⑥ Selects and invokes appropriate tool
⑦ Tool result stored in Memory System (short-term + long-term)
⑧ Reasoning continues until goal achieved
⑨ LLM provider (OpenAI/Claude/Groq) assists with reasoning
⑩ Final output generated and returned to API
⑪ Frontend updates in real-time via WebSocket
⑫ Metrics collected for monitoring

Component Dependencies

🔗 Dependency Hierarchy (Top to Bottom):
  • Layer 1 (Client): Web Browser, CLI, External Systems
  • Layer 2 (API): FastAPI Router, WebSocket Handler
  • Layer 3 (Services): Task Service, Agent Service, Tool Service
  • Layer 4 (Core): Agent, Reasoning, Memory
  • Layer 5 (Protocol): MCP Client, MCP Server
  • Layer 6 (Integration): LLM Clients, Tool Implementations
  • Layer 7 (Storage): PostgreSQL, Redis
  • Layer 8 (Monitoring): Prometheus, Grafana
📦 Key Integration Points:
  • Agent ↔ Reasoning: Task execution and strategy
  • Reasoning ↔ LLM: Decision making and planning
  • Agent ↔ MCP Client: Tool discovery and invocation
  • MCP Client ↔ MCP Server: Protocol communication
  • Agent ↔ Memory: Context storage and retrieval
  • Memory ↔ Database: Persistence and caching
  • Agent ↔ API: Task results and status
  • API ↔ Frontend: Real-time updates via WebSocket
